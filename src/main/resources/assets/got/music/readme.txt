Музыкальные пакеты - это аналог ресурспаков. Это тоже .zip-архив с содержимым, но место текстур он добавляет музыку, которая играет в биомах. 
Соответственно, кладутся пакеты с музыкой не в .minecraft/resourcepacks, а в .minecraft/musicpacks.

Внутри пакета должна быть папка assets, а в ней папка gotmusic. Туда вы будете класть треки.
Все треки должны быть в формате .ogg, а имена должны быть простыми и понятными, без всяких спецсимволов.

Теперь перейдём к самому сложному: как заставить нужный трек играть в нужном биоме.
Создаём файл "music.json" и кладём его рядом с папкой assets, которую мы создали в прошлом абзаце.

В этом файле вы определяете все треки, их имена и регионы, в которых они будут воспроизводиться, а также другие свойства:

- "tracks": массив объектов, по одному для каждого трека.
	- "file": название трека вместе с его расширением, например Test.ogg.
	- "title": красивое название трека, отображаемое в игре. Не обязательно.
	- "regions": континенты, в которых может воспроизводиться трек.
		- "name": название континента. Чтобы музыка играла везде, укажите регион "all".
		- "sub": биомы, в которых играет музыка. Уточняет музыку континента. Не обязательно.
		- "categories": обстоятельства, при которых трек может воспроизводиться в этом регионе. Не обязательно.
		- "weight": процент вероятности воспроизведения. У любимого трека ставьте больше. Не обязательно.
	- "authors": копирайт, который никого никогда не интересует. Но вы можете его добавить, убив время)

Например, вот мой music.json (ниже). Он заставляет играть файл sus.ogg в биоме Долины Аррен и днём, и ночью. Автором трека является амогус, все авторские права защищены.

{
  "tracks": [
    {
      "file": "sus.ogg",
      "title": "SUS",
      "regions": [
        {
          "name": "westeros",
          "sub": [
            "arryn"
          ],
          "categories": [
            "day",
            "night"
          ]
        }
      ],
      "authors": [
        "AMOGUS"
      ]
    }
  ]
}


Если вы не знакомы с тем, как правильно форматировать JSON, ищите примеры, ищите учебные пособия или попросите кого-нибудь о помощи. Я уверен, что у вас всё получится!
